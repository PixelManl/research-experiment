# 改完 / 修 bug 后

1. **对照 plan**  
   - 实现清单是否 1:1 勾完？有无「顺手改」未写入 plan？

2. **联动**  
   - 组件组是否半更新？测试/最小检查是否过？

3. **结果链（防 7-01 误用）**  
   - 新 run 写入 `runs/<slot>/<日期>/...`。  
   - 更新 `runs/<slot>/index.md`：**当前** 指向新 run；旧错误 run → **废弃**。  
   - 凡声明来自废弃 run 的 **data** → 删除或标废弃；A→B→C→D 从错误点 B 起向下游清理。  
   - 作图/写结论 **只引用当前**；禁止「日期更新所以对」。

4. **index**  
   - `scripts` / `runs` / `data` 的 index 是否仍短且准？

5. **回写 main？**  
   - 仅当目标本身变了；过程细节留 plan 与 runs index。
