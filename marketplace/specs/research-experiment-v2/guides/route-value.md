# Route Value：做完 ≠ 有价值

## 一句话

**阶段产物都对，仍可能不回答原始研究问题；禁止「上阶段做完所以继续堆下一阶段」。**

## 继续加阶段 / 加工程前（短清单）

- [ ] **原始价值问题**是什么？（一句话）  
- [ ] 当前结果能否 **判别**「路线成立 / 不成立」？  
- [ ] 相对 **最强简单 baseline** 的最小判别实验是什么？做了吗？  
- [ ] 若只是更精确、更多基础设施、更复杂协议：是否在 **逃避** 价值问题？  

## 禁止

- 用「实现正确 / 测试通过 / 图更好看」替代价值判断。  
- No-Go 或价值不足后，不经重新陈述价值问题就上更复杂修复路线。  
- 把 route 修成「完美工程」却永远不碰判别实验。

## 与 organize / intent

- 文件系统组织对了 ≠ 科学路线对了。  
- plan 实现清单勾完 ≠ 值得写进论文叙事。  
- 价值判断属 **T3（人定）**；Agent 应提出疑问与最小判别实验，不擅自宣布路线胜利。
